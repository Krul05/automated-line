# Montrac web service

Веб-сервис управляет IRM-станциями Montrac через Moxa Virtual COM ports.

Подробная документация по запуску, настройке и эксплуатации находится в [docs/service.md](docs/service.md).
Подробное описание API с примерами запросов находится в [docs/api.md](docs/api.md).

## Что Делает

- Кнопка `Станции 2 и 4 · 3 с`: тележки автоматически отпускаются сразу на остальных станциях, а на станциях с номерами 2 и 4 ждут 3 секунды.
- Кнопка `Все станции · 3 с`: каждая станция держит тележку 3 секунды.
- Кнопка `Запустить` внутри карточки станции отправляет `START FORWARD` только для тележки на этой станции.
- Режимы работы создаются и удаляются через интерфейс. В каждом режиме задаётся задержка в секундах для каждой станции.
- Перед отправкой `START FORWARD` сервис проверяет, что в перегоне до следующей станции сейчас 0 тележек. После отправки команды тележка добавляется в этот перегон до прихода на следующую IRM-станцию.
- В интерфейсе можно добавлять, удалять, переименовывать станции и менять COM-порты. Изменения сохраняются в рабочий `config.json`; базовый `config.example.json` не меняется.

## Запуск

1. Установите зависимость:

   ```powershell
   pip install -r requirements.txt
   ```

2. Скопируйте пример конфигурации при необходимости:

   ```powershell
   copy config.example.json config.json
   ```

   Если `config.json` отсутствует, сервис при первом запуске сам создаст его из `config.example.json` или из встроенной конфигурации COM1-COM6.

3. Запустите сервис:

   ```powershell
   python main.py --config config.json --host 0.0.0.0 --port 8080
   ```

4. Откройте:

   ```text
   http://localhost:8080
   ```

Для проверки интерфейса без реальных COM-портов:

```powershell
python main.py --mock --port 8080
```

Проверка логики без оборудования:

```powershell
python -m unittest discover -v
```

## Формат IRM-команд

Сервис использует тот же формат, что и рабочие скрипты:

```text
01 group id_high id_low command checksum 03
```

`checksum` считается как `group XOR id_high XOR id_low XOR command`.

Используемые коды:

- `83`: присутствие тележки, сообщение приходит от IRM.
- `51`: `START FORWARD`, команда отправляется в IRM.

## Настройки

`loop: true` означает маршрут `COM1 -> COM2 -> COM3 -> COM4 -> COM5 -> COM6 -> COM1`.
Если линия не замкнутая, поставьте `loop: false`; тогда последняя станция не будет отпускать тележку дальше автоматически.

Редактировать конфигурацию через интерфейс можно только когда линия пустая: нет занятых станций и занятых перегонов. Это защищает сервис от потери текущей карты движения тележек.

Пример режима:

```json
{
  "id": "stop_all",
  "name": "Все станции · 3 с",
  "stationDelays": {
    "1": 3,
    "2": 3,
    "3": 3
  }
}
```

Ключи в `stationDelays` — номера станций, значения — задержка в секундах. `0` означает отпустить тележку сразу, если в следующем перегоне сейчас 0 тележек.
